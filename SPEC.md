# ひかるのやることリスト 仕様書 v0.3

階層型タスク管理アプリ。フォルダ → ゴール → ステップ → Todo の4階層(フォルダは任意)を一目で俯瞰でき、
スマホ・デスクトップのホーム画面からワンタップでTodoに到達できる。

## フォルダ階層(v1.2で追加)

ゴールを「しまう」任意の最上位階層。例:「3年前期期末テスト」フォルダに科目ごとのゴールを入れる。

- フォルダは**任意**。フォルダに属さないゴールは従来通りトップレベルに表示(既存データはそのまま)
- フォルダの進捗 = 配下ゴール進捗の平均(ゴール0件なら0%)
- フォルダは折りたたみ可能。タイトル+進捗バーのみのシンプルな見た目(期限なし・説明なし)
- ゴールは「ゴールメニュー → フォルダへ移動」でフォルダに出し入れできる
- フォルダ削除時: 配下ゴールは削除せずトップレベルに戻す(データを消さない)

## 現在ステップへのフォーカス(v1.2で追加)

- 「現在のステップ」= ゴール内で sortOrder が最初の未完了ステップ
- ツリービュー: 現在のステップに **NOW** バッジ(ネオン発光)を付けて展開表示。それ以降の未来ステップは折りたたみデフォルト(タップで開ける)。完了ステップも折りたたみ(既存挙動)
- Todoビュー: デフォルトは各ゴールの**現在ステップのTodoのみ**表示。ヘッダーに「すべて表示」トグルがあり、ONで全未完了Todo表示(done扱いステップのTodoは常に除外)
- ステップ完了で次のステップが「解放」される(Todoビューに次ステップのTodoが現れる)

## デザイン原則(最重要)

1. **順番が見た目で分かる**:ステップには大きな番号(STEP 1, 2, 3…)を振り、上から下に進む流れが一瞬で読み取れる
2. **進捗が見た目で分かる**:数字より先にバー・チェックの視覚情報で伝える(%表記は補助)
3. **文字数を絞る**:一画面に出す文字を最小限に。長いタイトルは省略表示、説明文は原則持たない(タイトルだけで運用)
4. 余白を多く、1項目1行。「パッと見て敬遠されない」ことを機能追加より優先する

## カラーテーマ(確定):ダーク×ネオン

ゲーム感覚でタスクを消化する、達成感重視のスタイリッシュ路線。

| 役割 | 色 | 用途(60-30-10の法則) |
|------|-----|------|
| ベース(60%) | ダークグレー `#121212` / チャコール `#1E1E1E` | 背景・カード面 |
| メイン(30%) | ネオンパープル `#BB86FC` / ミントグリーン `#03DAC6` | ヘッダー、進捗バー、主要UI |
| アクセント(10%) | ビビッドピンク `#FF0266` | 追加ボタン等、最も目立たせる要素 |

**システムカラー(ステータス用に予約、メインカラーとは分離)**
- 完了: グリーン系(例 `#4CAF50`)
- 警告・期限切れ: レッド系(例 `#FF5252`)
- 保留・無効: グレー / イエロー系

演出: Todo完了時にチェックが光る・進捗バーがネオン発光(box-shadow)するなど、
「消化する快感」を小さく足す。ただし派手にしすぎず視認性優先。

## コンセプト

- **上から下へ**:ゴール(なぜやるか)→ ステップ(どう進めるか)→ Todo(今日何をするか)
- **下から上へ**:Todoを潰すとステップ・ゴールの進捗率が自動で上がる
- 「今日のTodo」への到達は常にワンタップ

## データモデル

```
folders(フォルダ)※v1.2追加
  id          uuid PK
  title       text
  sort_order  int
  created_at  timestamptz

goals(ゴール)
  id          uuid PK
  folder_id   uuid FK → folders, nullable  -- NULL = トップレベル(v1.2追加)
  title       text
  sort_order  int
  status      text  -- active / done / archived
  due_date    date nullable  -- NULL = 無期限
  created_at  timestamptz

steps(ステップ)
  id          uuid PK
  goal_id     uuid FK → goals
  title       text
  sort_order  int
  status      text  -- active / done
  created_at  timestamptz

todos(Todo)
  id           uuid PK
  step_id      uuid FK → steps
  title        text
  done         boolean
  sort_order   int
  created_at   timestamptz
  completed_at timestamptz nullable
```

- 進捗率は保存せず計算で出す。**計算ルール(v1.1で修正)**:
  - ステップ進捗 = status が done なら 100%。それ以外は配下Todoの完了率(Todoが0件なら 0%)
  - ゴール進捗 = 全ステップ進捗の平均(ステップ0件なら 0%)
  - → Todoを全部消化しても、Todoが空のステップが残っていれば100%にならない
  - ステップは手動でも「完了」にできる(Todoなしのステップを消化するため)。全Todo完了時の自動doneも維持
- Todoは必ずステップに属する(階層を崩さない)。「その他」ステップを作る運用でカバー

## 画面(v1は2画面のみ)

### 1. ツリービュー(俯瞰)
- ゴールをカードで縦に並べ、各カードに進捗バー(%は補助表示)
- 期限ありゴールは「残り○日」を進捗バー横に表示。無期限は何も出さない(期限切れは赤字)
- カードを展開するとステップ(STEP番号つき)→ Todo がツリー表示
- その場で追加・編集・チェック・削除ができる(モーダル遷移を極力なくす)
- 完了したTodoは薄く表示、完了ステップは折りたたみ

### 2. Todoビュー(実行)
- 未完了Todoだけをフラットに一覧表示(所属ゴール/ステップ名を小さく併記)
- チェックで即完了 → リストから消える(進捗に即反映)
- **スマホでPWAを開いたときのデフォルト画面はこちら**。デスクトップはツリービューがデフォルト

画面下部タブ(またはヘッダー)で2画面を切り替え。

## JSONインポート(v1.1で追加)

チャットのClaudeにタスク分解させた結果を貼り付けて、ゴール一式を一括登録する機能。

- ツリービューに「JSONインポート」ボタン → テキストエリアに貼り付け → 内容のプレビュー(ゴール○件/ステップ○件/Todo○件)→ 取り込み
- 不正なJSONや形式違いはエラーメッセージ表示(登録はしない)
- 形式(正式フォーマット):

```json
{
  "folders": [
    {
      "title": "フォルダ名(例: 3年前期期末テスト)",
      "goals": [
        {
          "title": "ゴール名",
          "dueDate": "2026-08-31",
          "steps": [
            { "title": "ステップ名", "todos": ["Todo1", "Todo2"] },
            { "title": "Todoなしステップ", "todos": [] }
          ]
        }
      ]
    }
  ],
  "goals": []
}
```

- `folders` と `goals` はどちらも省略可(どちらか一方だけでよい)。トップレベルの `goals` はフォルダなしのゴールになる
- 旧形式(`goals` のみ)もそのまま有効(後方互換)
- `dueDate` は `"YYYY-MM-DD"` または `null`(無期限)。省略も null 扱い
- `todos` は文字列配列。省略可
- folders/steps/goals の並び順 = 配列順(sortOrderに反映)
- 既存データには追記(上書きしない)
- フォルダ名が既存フォルダと完全一致する場合は新規作成せず、その既存フォルダにゴールを追加する

## ゴールの期限(確定)

- ゴール作成・編集時に「期限あり(日付選択)」か「無期限」を選べる
- 期限はゴール単位のみ(ステップ・Todoには持たせない。必要になったら追加)
- 表示:期限あり → 残り日数、期限超過 → 赤字で「○日超過」、無期限 → 非表示

## v2.1: ドラッグ&ドロップ(確定スコープ)

ツリービューで、各行の左端の**グリップ(⋮⋮)を掴んで**ドラッグする。行本体のクリック編集とは干渉させない。
スマホは**グリップを長押し(約250ms)**でドラッグ開始(通常スクロールと共存)。ドラッグ中は行が浮き上がり、挿入位置にネオンのインジケータ線を表示。

### 縦ドラッグ = 並び替え&親の変更
- フォルダ: トップレベル内で並び替え
- ゴール: 同じ場所での並び替えに加え、**フォルダの中/外へドロップで所属変更**(フォルダ領域内に落とすと入る、外に落とすと出る)
- ステップ: 同ゴール内の並び替え+**別ゴールへのドロップで移動**(配下Todoごと)
- Todo: 同ステップ内の並び替え+**別ステップへのドロップで移動**

### 横ずらし = 階層の昇格/降格(ドロップ時のX変位で判定、しきい値~40px)
- **Todo→左(アウトデント)**: 同ゴール内で新しいステップに昇格(タイトル引継ぎ、元ステップの直後に挿入)
- **ステップ→右(インデント)**: 直前のステップのTodoに降格(タイトル引継ぎ。配下Todoも同じステップへ順序維持で移動)。直前にステップがない場合は無効
- **ステップ→左(アウトデント)**: ゴールに昇格(タイトル引継ぎ、同フォルダ/トップレベルの元ゴール直後に挿入。配下Todoは新ゴール内の「その他」ステップ1つにまとめて移動。Todoが無ければステップなしの空ゴール)
- **ゴール→右**: 直前(上)にあるフォルダに入る。直前がフォルダ(またはフォルダ内ゴール)でなければ無効
- **ゴール→左**: フォルダから出てトップレベルへ
- 無効な操作はドロップ時に元の位置へスナップバック(揺れアニメーションで否定を伝える)

### 実装メモ
- sortOrderの振り直しは移動先の兄弟間で再採番。親変更は folderId / goalId / stepId の付け替え
- 保存はRepository経由でバッチ(Supabaseモードでも1操作=まとめて保存、失敗時はリロードで整合)
- 進捗・NOWバッジ・Todoビューは移動後の構造から自動再計算(既存ロジックのまま)
- prefers-reduced-motion時はアニメーション最小化

## v3(確定スコープ)

### 削除の確認(インライン2段階)
- ×ボタンを押すと、その場でボタンが「削除?」(赤・小)に変わり、2.5秒以内にもう一度押すと削除。時間切れ/他所タップで元に戻る
- window.confirm は全廃(フォルダ削除含めこの方式に統一)
- 対象: Todo/ステップ/ゴール/フォルダの全削除ボタン

### アンドゥ(Ctrl+Z)
- データ変更(追加・編集・削除・チェック・D&D移動・インポート)前に状態スナップショットを保存(深さ20)
- Ctrl+Z / Cmd+Z で1つ戻す。差分(作成/更新/削除)を計算してRepository経由で適用(ID保持で復元、Local/Supabase両対応)
- スマホ用: 削除・移動の直後に「元に戻す」ボタン付きトースト(4秒)を表示
- タブ復帰の自動再取得時はスタックをクリア(別端末の変更との整合のため)

### ウィジェットビュー
- URL `?widget=1` で起動する超軽量表示: 「今日のやること」ヘッダー+現在ステップの未完了Todoリストのみ(チェック可、音あり)。タブ・追加UI・装飾なし、コンパクト行
- 5分ごと+フォーカス時に自動再取得(Supabaseモード)
- manifest に shortcuts を追加(Android: アイコン長押し→「今日のやること」)
- デスクトップ: 小窓ショートカット「ひかるのやることリスト ウィジェット.lnk」(Edge --app --window-size=380,520)を作成
- アプリバッジ: navigator.setAppBadge(未完了Todo数)を対応環境で(非対応は黙ってスキップ)

### 記録タブ(統計+実績棚)
- タブバーに3つ目「記録」を追加
- 上段: ストリーク大表示+過去7日の完了数バーグラフ(CSS/自作、ネオン)
- 下段: 「実績の棚」= 完了/アーカイブしたゴールをトロフィーカード(達成日・消化Todo数)で並べる。ゴールを「完了」にする動線もここから見えるように
- データは既存の completedAt / status から計算(スキーマ変更なし)

### ステップとTodoの期限(v3.1)
- StepとTodoの両方に任意の `dueDate`(YYYY-MM-DD | null)を追加。DB: `steps.due_date date` / `todos.due_date date` いずれもnullable(migration-v3.1.sql)
- 設定UI: 行の小さなカレンダーアイコン → インラインで日付選択/「無期限」でクリア(ゴールの期限UIと同じ流儀)。ステップはヘッダー行、Todoは各行
- 表示: 行の右に小さなチップ「残り3日」「今日」「明日」/期限切れは赤「○日超過」/無期限は非表示。完了済み(done)の行はチップを薄いグレー(赤くしない)
- Todoビューとウィジェットの並び順: dueDate昇順(期限切れ→今日→近い順)、無期限は最後(その中では作成順)。ツリー内の並び(ステップ・Todo)は従来通りsortOrder(D&D)
- インポート: steps/todos配列の要素にオブジェクト形式で `dueDate` を任意指定可(todos は従来の文字列も引き続き受理。例: `{ "title": "...", "dueDate": "2026-07-25" }`)。エクスポートはdueDateありの行をオブジェクト形式で出力
- Undo・進捗計算・ストリーク・NOW判定への影響なし(dueDateは表示と並び順のみ)

### v3に入れないもの
- プッシュ通知(iOS PWAの制約+サーバー必須のため見送り継続)
- 本物のOSウィジェット(PWA不可。ネイティブ化するときに再検討)

## v1に入れないもの(候補として記録)
